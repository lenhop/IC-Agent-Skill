## Task Assignment Principles

1. Kiro does NOT write code or run tests unless explicitly required by the Boss
2. Every task MUST produce a report in a `.md` file — mandatory, no exceptions
3. Developers (Claude Code, VSCode, Trae) handle all implementation work
4. Cursor handles all testing, validation, and quality assurance
5. Kiro reviews all reports and verifies alignment with requirements
6. Kiro evaluates team member performance and adjusts assignments accordingly
7. Tasks are assigned based on complexity, capability, and current workload

---

## Report Requirement (MANDATORY)

Every task assigned to any team member MUST result in a report file:

- **Location:** `{project}/tasks/`
- **Naming:** `YYYYMMDD-{task-id}-{task-name}-{executor}-rpt.md`
- **Required sections:** Status, Files Created/Modified, Results, Issues/Notes

No task is considered complete without a submitted report.
