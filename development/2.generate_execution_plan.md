# 版本 3：全面版（Comprehensive / Full）

## 英文组

Act as an IT architect and AI expert with 10 years of experience in top tech companies. Generate a complete, production‑ready 3‑phase task breakdown and execution plan following these strict rules:

### Core Mission

1. Based on task analysis and optimization results, divide the entire project into **3 standard phases**:

   Phase 1 (Core Functions) → Phase 2 (Optimization & Enhancement) → Phase 3 (Testing & Acceptance).

2. For each phase, define all of the following in detail:

   - **Specific task content**: detailed to file / module / function level.
   - **Deliverables**: code files, configuration files, documents, test cases, scripts, etc.
   - **Acceptance criteria**: **clear, verifiable, testable, no ambiguous expressions**.
   - **Estimated working hours**: calculated in person-days per phase.

3. Prioritize all key tasks by business value and risk to ensure core capabilities are delivered first.

### Output Requirements

1. Use standard Markdown formatting with strict hierarchical structure:
   1. XX → 1.1 XX → (1) XX → ① XX.
2. Highlight all key information with **bold**, especially acceptance criteria and deliverables.
3. Ensure the final execution plan is **directly usable for development scheduling, implementation, and management review**.
4. Keep the output concise, structured, and focused; avoid repetition, verbosity, and irrelevant descriptions.

## 中文组

作为具有 10 年顶级科技公司经验的 IT 架构师和 AI 专家，按以下严格规则生成完整、可落地的三阶段任务拆分与执行计划：

### 核心目标

1. 基于任务分析与优化结果，将整个项目拆分为**标准 3 阶段**：

   阶段 1（核心功能）→ 阶段 2（优化增强）→ 阶段 3（测试验收）。

2. 每个阶段详细定义以下内容：

   - **具体任务内容**：细化到文件 / 模块 / 函数级别。
   - **交付物**：代码文件、配置文件、文档、测试用例、脚本等。
   - **验收标准**：**清晰、可验证、可测试、无模糊表述**。
   - **预估工时**：按阶段以人天为单位核算。

3. 按业务价值与风险对所有关键任务排序，确保核心能力优先交付。

### 输出要求

1. 使用标准 Markdown 格式，严格层级结构：
   1. XX → 1.1 XX → (1) XX → ① XX。
2. 所有关键信息使用**加粗**突出，尤其验收标准与交付物。
3. 确保最终执行计划**可直接用于开发排期、落地执行与管理评审**。
4. 输出简洁、结构化、聚焦要点；避免重复、啰嗦与无关描述。
