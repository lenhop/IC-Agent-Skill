Core Mission

- Based on the results of task analysis and optimization, divide the task into 3 phases: Phase 1 (Core Functions) → Phase 2 (Optimization & Enhancement) → Phase 3 (Testing & Acceptance).
- For each phase, clearly define:
  - Specific task content (detailed to file/module level).
  - Deliverables (e.g., code files, configuration files, test cases).
  - Clear acceptance criteria (verifiable, no ambiguous expressions).
  - Estimated working hours (by phase, in person-days).
- Prioritize key tasks to ensure core capabilities are delivered first.



Output Requirements

- Use Markdown formatting (headings, lists, tables, code blocks) with a clear hierarchical structure: 1. XX → 1.1 XX → (1) XX (easy to read).
- Highlight key information (e.g., acceptance criteria) with bold formatting.
- Ensure the execution plan can be directly used for development scheduling and implementation.
- Keep the output concise (focus on key points, avoid repetition and verbosity).



1. 核心使命

   - 基于任务分析与优化的结果，将任务划分为 3 个阶段：阶段 1（核心功能）→ 阶段 2（优化与增强）→ 阶段 3（测试与验收）。

   - 为每个阶段明确界定：

     - 具体任务内容（细化到文件 / 模块粒度）。
     - 产出物（如代码文件、配置文件、测试用例）。
     - 明确的验收标准（可验证、无模糊表述）。
     - 预估工时（按阶段，以人天计）。

   - 优先安排关键任务，确保核心能力先交付。

   

2. 输出要求

   - 使用 Markdown 排版（标题、列表、表格、代码块），层级结构清晰：1. XX → 1.1 XX → （1）XX（清晰易读）。
   - 对关键信息（如验收标准）使用加粗格式突出。
   - 执行计划须可直接用于开发排期与实施。
   - 输出保持简洁（聚焦要点，避免重复赘述）。
