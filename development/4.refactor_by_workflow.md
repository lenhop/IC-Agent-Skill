

##### Core Competencies

1. **Business-Driven Code Grouping**

   Group scattered functions by business workflow/functional modules to ensure logical cohesion, aligning code structure with business logic.

2. **Modular Encapsulation**

   Encapsulate related function groups into independent classes, prioritizing @classmethod for consistent coding style and enhanced modularity.

3. **Service Interface Standardization**

   Assemble external-facing services using refactored classes/methods, exposing unified, concise interfaces to reduce integration/invocation costs.

4. **Logic & Behavior Preservation**

   Ensure 100% consistency of original business logic; refactored code must be maintainable, extensible, and human-readable.

5. **Exception & Logging Best Practices**

   Reject null returns in exception handling (raise specific errors for proper propagation); add comprehensive logger statements to key code, exceptions, and errors for debugging/monitoring.

6. **Cross-File Dependency Management**

   After moving functions to new files, update upstream/downstream code to import implementations from new paths (eliminate import errors/missing modules).

7. **Code Quality Enforcement**

   - Add type hints for all parameters/return values, docstrings for classes/public methods, and comments for key code.
   - Remove duplicate code (extract common logic to reusable methods/base classes); follow single responsibility principle (small, focused functions/methods).
   - Comply with PEP8/project lint rules; sort/organize imports (remove unused ones); avoid hardcoding (use config files); standardize naming conventions for classes/methods/variables/files.
   - Validate data validity at method entry (raise clear errors); split deeply nested logic into smaller methods.

8. **Post-Refactoring Validation**

   Enforce testing after refactoring to ensure functional correctness; verify end-to-end project runnability.



##### Deliverables

- Complete, runnable refactored code (directly usable in projects).
- Clear documentation of all changes (e.g., file updates, import adjustments, logic splits).





##### 核心能力

1. **业务驱动的代码分组**

   按业务工作流 / 功能模块对分散的函数进行分组，确保逻辑内聚，使代码结构与业务逻辑对齐。

2. **模块化封装**

   将相关函数组封装为独立类，类内函数优先使用@classmethod 保证编码风格统一，提升模块化程度。

3. **服务接口标准化**

   基于重构后的类 / 方法组装对外服务，暴露统一、简洁的接口，降低集成与调用成本。

4. **逻辑与行为一致性保障**

   确保原有业务逻辑 100% 无变更；重构后代码需具备高可维护性、可扩展性和可读性。

5. **异常与日志最佳实践**

   异常处理拒绝返回 null 值（直接抛出特定错误保证错误正常传播）；在关键代码、所有异常 / 错误处添加完备的日志语句，支撑调试、监控与问题追踪。

6. **跨文件依赖管理**

   函数迁移至新文件后，更新上下游代码以从新路径导入实现（杜绝导入错误 / 模块缺失问题）。

7. **代码质量强制规范**

   - 为所有参数 / 返回值添加类型提示，为类 / 公共方法添加标准文档字符串，为关键代码添加注释。
   - 移除重复代码（将公共逻辑抽为可复用方法 / 基类）；遵循单一职责原则（保持函数 / 方法短小、职责聚焦）。
   - 符合 PEP8 / 项目 lint 规范；整理排序导入语句（删除未使用的导入）；使用配置文件管理常量（避免硬编码）；对类 / 方法 / 变量 / 文件采用统一命名规范。
   - 方法入口处校验数据有效性（抛出明确错误）；将深层嵌套逻辑拆分为更小的方法。

8. **重构后验证**

   重构完成后强制执行测试，确保功能正确性；验证项目端到端可正常运行。

   

##### 交付物

- 完整、可直接用于项目的可运行重构代码。
- 清晰的变更说明文档（如文件更新、导入调整、逻辑拆分等）。
