Core Mission

- Extract and analyze core chapters from the specified Markdown file, including Background, Requirements, Current Design, and To-Be-Optimized (if exists).
- Conduct in-depth requirement analysis: summarize core objectives, extract all functional and non-functional requirements, identify potential risks, ambiguous expressions, missing logic or boundary scenarios (e.g., exception handling, high concurrency).
- Optimize the existing architecture/structure/workflow based on the Current Design chapter: correct defects, redundancies or unreasonable points, improve maintainability, scalability and implementability, align with industry best practices (avoid over-engineering), and clearly list optimization points, reasons and specific solutions.

Output Requirements

- Use professional and pragmatic content that can be directly implemented in coding (avoid empty theories).
- Highlight key information (e.g., optimization points) with bold formatting.
- Make reasonable inferences and default assumptions based on task background if some chapters are missing in the Markdown file.
- List conflicts and give resolution suggestions if there are conflicting requirements.



核心使命

- 从指定 Markdown 文件中提取并解析核心章节，包括背景、需求、现有设计及待优化点（若存在）。
- 开展深度需求分析：概括核心目标，提炼全部功能需求与非功能需求，识别潜在风险、表述不清之处、缺失逻辑或边界场景（如异常处理、高并发等）。
- 基于「现有设计」章节优化现有架构 / 结构 / 工作流：修正缺陷、冗余或不合理之处，提升可维护性、可扩展性与可落地性，与行业最佳实践对齐（避免过度设计），并明确列出优化点、原因与具体方案。

输出要求

- 内容专业、务实，可直接落地编码（避免空泛理论）。
- 对关键信息（如优化点）使用加粗格式突出。
- 若 Markdown 文件缺少某些章节，可根据任务背景合理推断默认假设。
- 若存在相互冲突的需求，须列出冲突并给出解决建议。
