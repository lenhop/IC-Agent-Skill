# 版本 3：全面版（Comprehensive / Full）

Act as an IT architect and AI expert with 10 years of experience in top tech companies. Perform full-scale, production-grade refactoring by workflow with these strict rules:

1. Business-Driven Code Grouping

   Group scattered functions by business workflow/functional modules to ensure logical cohesion and align code structure with real business processes.

2. Modular Encapsulation

   Encapsulate related function groups into independent, single-responsibility classes; prioritize @classmethod for uniform style and strong modularity.

3. Service Interface Standardization

   Assemble external services using refactored components; expose stable, unified interfaces to reduce integration and invocation costs.

4. Logic & Behavior Preservation

   Ensure 100% consistency of original business logic; refactored code must be maintainable, extensible, and highly readable.

5. Exception & Logging Standardization

   Never return null in exceptions; raise clear, specific errors for proper propagation. Add log statements to key logic, exceptions, and error points.

6. Cross-File Dependency Management

   When moving code to new files, automatically update all upstream/downstream imports and references to ensure no broken dependencies.

7. Full Code Quality Enforcement

   - Add type hints, docstrings, and clear comments.
   - Remove duplicates; extract reusable methods and base classes.
   - Follow PEP8 / project lint rules; organize and clean up imports.
   - Use configuration instead of hardcoding; standardize naming.
   - Validate method inputs; split deeply nested logic.

8. Post-Refactoring Validation

   Perform testing after refactoring to ensure functional correctness; verify end-to-end project runnability.

9. Final Cleanup

   Delete duplicate, redundant, and zombie code after refactoring.

10. Deliverables

    Output complete, runnable, production-ready code and a clear list of all changes.

    

作为具有 10 年顶级科技公司经验的 IT 架构师和 AI 专家，按以下严格规则执行企业级全量按工作流重构：

1. 业务驱动代码分组

   按业务工作流与功能模块对分散函数进行分组，确保逻辑内聚，让代码结构与真实业务对齐。

2. 模块化封装

   将相关函数组封装为独立、单一职责的类；优先使用 @classmethod 保持风格统一与强模块化。

3. 服务接口标准化

   使用重构后的组件组装对外服务；提供稳定、统一的接口，降低集成与调用成本。

4. 逻辑与行为保真

   确保原有业务逻辑 100% 不变；重构后的代码必须可维护、可扩展、高可读。

5. 异常与日志规范化

   异常处理绝不返回 null；抛出清晰、明确的错误以保证正常传播。为关键逻辑、异常、错误点添加日志。

6. 跨文件依赖管理

   代码迁移至新文件后，自动更新所有上下游导入与引用，确保无依赖断裂。

7. 全量代码质量规范

   - 添加类型提示、文档字符串与清晰注释。
   - 移除重复代码；抽取可复用方法与基类。
   - 遵循 PEP8 / 项目规范；整理并清理导入。
   - 使用配置替代硬编码；统一命名规范。
   - 校验方法入参；拆分深层嵌套逻辑。

8. 重构后验证

   重构后必须执行测试，确保功能正确；验证项目端到端可正常运行。

9. 最终清理

   重构完成后删除重复、冗余与僵尸代码。

10. 输出物

    输出完整、可运行、可直接上线的代码，并提供清晰的变更清单。
