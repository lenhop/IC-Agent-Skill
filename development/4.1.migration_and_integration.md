# 版本 3：全面版（Comprehensive / Full）

## English

Act as an IT architect and AI expert with 10 years of experience in top tech companies. Optimize and refactor the internal service layer and external interface workflow with these strict rules:

1. ARCHITECTURE PRINCIPLE (MANDATORY)

   1. Internal storage module = internal upstream data source (single source of truth)
   2. Public interface module = ONLY unified public interface for data management and logic processing
   3. All downstream business modules CANNOT import the internal storage module directly
   4. All downstream modules MUST use the public interface module for ALL related data and logic operations

2. REFACTOR TASKS

   1. Move ALL logic processing and data-fetching logic from downstream files INTO the public interface module
   2. Keep only clean, unified, reusable functions/classes in the public interface module
   3. Remove ALL duplicate code (data query, logic processing, state handling, event appending)
   4. Ensure NO downstream files directly access the internal storage module
   5. Make the public interface module the single entry point for all related operations

3. CODE CHANGES

   1. Keep all existing behavior, business logic, and data types unchanged
   2. Keep the internal storage module as internal data storage only (no breaking changes)
   3. Expose clean, stable public APIs from the public interface module for downstream use
   4. Remove redundancy, duplication, and direct dependencies on internal storage from downstream

4. FINAL STATE

   1. Downstream modules import ONLY from the public interface module
   2. Internal storage module is called ONLY inside the public interface module
   3. All related logic and data operations are centralized in the public interface module
   4. Clean architecture, clear layers, highly maintainable, production-ready
   5. Add clear code comments, especially for the public interface modules

5. CLEANUP RULE

   1. After refactoring, delete all duplicate and redundant functions

1. 架构原则（强制）

   1. 内部存储模块 = 内部上游数据源（唯一真实数据源）

   3. 由公共接口模块对外提供简洁稳定的公共 API
   4. 移除冗余、重复逻辑及下游对内部存储的直接依赖
